This page demonstrates two ways to create responsive tables that transform from standard table layouts on desktop to card-based layouts on mobile devices:

1. **ResponsiveTable Component** - A reusable Astro component with explicit configuration
2. **Enhanced Markdown Tables** - Standard markdown tables with automatic responsive enhancement

Both approaches use **CSS-only transformation** with no JavaScript-based layout switching, preserving table semantics for screen readers at all breakpoints.

## How It Works

### Desktop View (> 640px)
- Tables display in standard tabular format
- Column headers visible in `<thead>`
- Normal row and column alignment

### Mobile View (â‰¤ 640px)
- Each table row transforms into a "card"
- Column headers are visually hidden but remain accessible to screen readers
- Each cell displays with a label (from the header) and its value
- Cards are stacked vertically for easy scrolling

### Accessibility Features

- âœ… Table semantics preserved at all breakpoints

### CSS-Only Transformation

The responsive transformation uses:
- `display: block` to reflow table elements on mobile
- `position: absolute` with clip technique to hide headers accessibly
- `::before` pseudo-element with `content: attr(data-label)` for labels
- Flexbox for label/value layout in card cells

## Best Practices

### When to Use This Pattern

âœ… **Good use cases:**
- Reference tables with 2-6 columns
- Command lists and API references
- Feature comparison tables
- Configuration options

âŒ **Not recommended for:**
- Large data tables (> 8 columns)
- Tables requiring horizontal scrolling on desktop
- Complex nested table structures
- Tables with merged cells

### Content Guidelines

1. **Keep headers concise** - They become labels on mobile
2. **Limit columns** - 2-6 columns work best
3. **Use semantic data** - Numbers, dates, status indicators
4. **Avoid very long content** - Cards grow vertically on mobile

## Limitations

- **JavaScript dependency for markdown tables**: Standard markdown tables require the client-side script to add `data-label` attributes
- **Wide content**: Extremely wide cell content may cause layout issues on mobile
- **Complex tables**: Merged cells and nested tables are not supported
- **Print styles**: Card layout may not be ideal for printing (consider print-specific CSS)

## Testing Responsive Behavior

To test the responsive transformation:

1. **Resize browser window** to below 640px width
2. **Use browser DevTools** device emulation (mobile viewport)
3. **Test on actual mobile devices** for real-world validation
4. **Use screen reader** to verify accessibility at all breakpoints
